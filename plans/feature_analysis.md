# 《末光咏叹》现存缺失的“必备”功能分析

通过对目前项目的代码（HTML, Data, Engine, Main, UI）进行全面的审阅，目前《末光咏叹》已经具备了相当成熟的底层框架（包括完善的GCD/oGCD战斗排轴、多维度的装备生成与强化精炼、全属性词条等）。
但如果以一款**成熟的、长线可玩的“放置类 RPG (Idle RPG)”**作为标准，目前系统仍然缺失以下几个**“必备”**的体验与玩法功能：

## 1. 核心机制层缺失
*   **真实离线收益 / 后台挂机计算 (Offline Progress & Background Tick)**
    *   **现状：** 目前游戏依赖 `requestAnimationFrame` 进行战斗循环。如果玩家切换浏览器标签页或者关闭浏览器，时间轴会冻结或在重返时出现巨大的 `deltaTime` 跳跃（容易导致系统状态异常）。
    *   **必备性：** 放置游戏的灵魂。需要实装：关闭页面记录时间戳，再次打开时计算时间差，并通过“快速模拟战斗”或“按挂机公式结算（基于挂机前的 DPS 和胜率）”直接给予玩家离线经验、金币与掉落。
*   **敌方技能与机制 (Enemy AI & Skills)**
    *   **现状：** `Monster` 类目前只是一个“数值沙袋”，所谓的Boss（逢10层的领主）也仅仅是血量和攻击力的倍率更高，没有任何其他机制。
    *   **必备性：** 当前玩家可以排极其深度的技能轴，但这套排轴系统其实是“打木桩”。如果Boss会释放AOE、眩晕玩家、开启反伤盾、给自己回血，那玩家就需要针对不同的Boss编排不同的“对策轴”（例如留着打断技能或爆发技能）。

## 2. 经济与成长层缺失
*   **金币回收系统：商店/黑市 (Shop / Black Market)**
    *   **现状：** 金币目前唯一的用途是“强化装备”和“营地里花 50G 回血”。在游戏中后期，金币必然会出现严重溢出。
    *   **必备性：** 需要一个消耗金币的系统。例如：提供一个商店购买随机盲盒装备（类似暗黑的赌博）、购买特殊的强化保护符、购买遗漏的宝珠，或者购买永久的账号被动属性。
*   **转生/轮回机制 (Prestige / Rebirth)**
    *   **现状：** 玩家只有单一维度的往上推塔爬层，数值会呈指数级无限膨胀（即使有经验惩罚和软上限）。如果遇到彻底推不动的数值墙，玩家只能原地无意义刷怪。
    *   **必备性：** 引入“轮回/转生”概念（重置等级、楼层和普通装备），保留核心的“终焉精华”或转化为特定的“魂石”，用来点亮永久的【天赋树】或【全局乘区】。这能赋予游戏多周目的重复游玩价值。

## 3. 数据反馈与体验层缺失 (QoL)
*   **战斗统计 / DPS 面板 (DPS Meter)**
    *   **现状：** 有细致的文字战斗日志，但玩家无法直观得知“我这套排轴到底秒伤（DPS）是多少？”、“哪个技能占比最高？”
    *   **必备性：** 对于这种注重 Build 和排轴的游戏，DPS 统计面板（甚至包括治疗量、护盾承伤）是极其核心的验证工具，方便玩家直观地比对哪套轴输出更高。
*   **动态的技能描述 (Dynamic Tooltips)**
    *   **现状：** `data.js` 里的技能描述 `desc` 都是静态写死的，比如“造成220%伤害”。
    *   **必备性：** 在UI展示上，技能说明应当动态换算。结合玩家当前面板，直接显示出“造成 4,500K 点伤害”，这样玩家在排轴时心里才有底。

## 4. 其他次要但也极度关键的功能
*   **音效反馈 (SFX/BGM)：** 攻击、暴击、掉落（特别是终焉极品掉落）时的声音反馈。
*   **进阶自动拾取过滤 (Advanced Loot Filter)：** 目前只有一个下拉框选择丢弃的品质，高端放置游戏通常允许设置“只保留带特定词条的首饰”等规则。
