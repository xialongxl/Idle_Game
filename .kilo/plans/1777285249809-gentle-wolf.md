## 二、engine.js — 三处改动

### 2.1 effectHandlers.dot — 传递 stateName/stateEmoji/totalDur

当前（第328行）：
```js
dot: (e, engine) => { engine.mob.buffs.push(e); return '附加[暗系减益] '; },
```

改为：
```js
dot: (e, engine) => {
  e.stateName = e.stateName || 'DoT';
  e.stateEmoji = e.stateEmoji || '☠️';
  e.totalDur = e.dur;
  e.tickTimer = 0;
  engine.mob.buffs.push(e);
  return `附加[${e.stateName}] `;
},
```

**注意**：`e` 已由 `executeSkill` 中 `JSON.parse(JSON.stringify(eff))` 深拷贝，所以直接在 `e` 上赋值是安全的。追加 `tickTimer: 0` 初始化确保后续 processEffects 中 `eff.tickTimer` 不会是 undefined。

### 2.2 effectHandlers.channel_immune — 删除硬编码 dot

当前（第371-374行）中 handler 额外 push 了一个硬编码 dot，与 s43 effects 数组中的标准 dot 重复：

```js

  engine.player.buffs.push({ type: 'channel_immune', dur: e.dur });
```

改为删除硬编码 dot（s43 的 effects 数组中已有标准 `{type:'dot',dur:8000,dps:4.0}` 会走 `effectHandlers.dot` 处理，无需重复 push）：

```js
channel_immune: (e, engine) => {
```

### 2.3 processEffects — DoT 日志格式替换

当前（第1226行）：
```js
this.log(`☠️ [流血跳跃] 造成 ${formatNumber(res.val)} 点伤害`, 'sys');
```

改为（区分星光虹吸和其他 DoT）：
```js
let dotLogMsg = `${eff.stateEmoji || '☠️'} [${eff.stateName || 'DoT'}] 造成 ${formatNumber(res.val)} 点伤害`;
if (eff.stateName === '虹吸') {
  let healAmt = Math.floor(res.val * 0.3);
  this.player.hp = Math.min(this.player.getMaxHp(), this.player.hp + healAmt);
  dotLogMsg += `，汲取 ${formatNumber(healAmt)} 点生命`;
```

**关于星光虹吸的汲取机制**：原文要求日志格式为 `🩸 [虹吸] 造成 X 点伤害，汲取 Y 点生命`，但当前 engine 代码中并未实现星光虹吸的汲取效果（仅是一个纯 DoT）。需确认：是仅改日志文案还是实际添加汲取生命逻辑？

**决策（已确认）**：添加实际汲取逻辑，按 DoT 伤害的 30% 回复玩家生命，日志和实际效果一致。

**HoT 日志保持不变**：第1196行 `💚 [持续恢复] 恢复 X 点生命` 不动。

### 2.4 删除硬编码术语

删除 `☠️ [流血跳跃]` 等硬编码文案，统一使用动态的 `${eff.stateEmoji} [${eff.stateName}]` 格式。

## 三、ui.js — 重写 mob-buffs 渲染逻辑

当前（第126-128行）：
```js
document.getElementById('mob-buffs').innerHTML = m.buffs
  .map(b => `<div class="buff-icon" title="${b.type}">${b.type === 'dot' ? '☠️' : '📜'}</div>`)
  .join('');
```

改为完整的 DoT 环形倒计时渲染：

```js
document.getElementById('mob-buffs').innerHTML = m.buffs.map(b => {
  if (b.type === 'dot') {
    let emoji = b.stateEmoji || '☠️';
    let name = b.stateName || 'DoT';
    let remaining = Math.ceil(b.dur / 1000);
    let dpsPct = (b.dps * 100).toFixed(0);
    let titleText = `${name}：每秒造成${dpsPct}点伤害，剩余${remaining}秒`;
    if (name === '虹吸') titleText += '，汲取生命';
    let isLow = remaining <= 3;
    let progress = b.totalDur > 0 ? b.dur / b.totalDur : 1;
    let textCls = isLow ? 'dot-timer-text dot-timer-low' : 'dot-timer-text';
    let rectW = 52;
    let rectH = 24;
    let rx = 6;
    let ry = 6;
    return `<div class="buff-icon" title="${titleText}">
      <svg class="buff-ring" viewBox="0 0 ${rectW} ${rectH}" preserveAspectRatio="none">
        <rect class="buff-ring-bg" x="1" y="1" width="${rectW-2}" height="${rectH-2}" rx="${rx}" ry="${ry}" />
        <rect class="buff-ring-fg" x="1" y="1" width="${rectW-2}" height="${rectH-2}" rx="${rx}" ry="${ry}" data-progress="${progress}" />
      </svg>
      <span class="dot-emoji">${emoji}</span>
      <span class="${textCls}">${remaining}</span>
    </div>`;
  }
  return `<div class="buff-icon" title="${b.type}">📜</div>`;
}).join('');

// 动态设置 stroke-dashoffset（需获取路径总长度）
requestAnimationFrame(() => {
  document.querySelectorAll('.buff-ring-fg').forEach(el => {
    let totalLen = el.getTotalLength();
    el.style.strokeDasharray = totalLen;
    let progress = parseFloat(el.dataset.progress) || 1;
    el.style.strokeDashoffset = totalLen * (1 - progress);
  });
});
```

**关键技术点**：
- SVG 使用 `preserveAspectRatio="none"` 让矩形路径可以拉伸填充任意宽高的容器
- `stroke-dasharray` 和 `stroke-dashoffset` 通过 JS 动态计算（需 `getTotalLength()`）
- 前景环初始 `stroke-dashoffset` 不可设为0（否则会闪烁），用 `visibility:hidden` 或极大值初始化，在 JS 计算后再显示
- `data-progress` 属性保存进度值供 JS 读取

## 六、风险评估

- **SVG `getTotalLength()` 兼容性**：所有现代浏览器均支持，IE 不支持（但该项目已使用 ES6 module，无需考虑 IE）
- **SVG `preserveAspectRatio="none"`**：可能导致 stroke 粗细随容器宽高比变化而变形。若效果不理想，可改用固定 viewBox 尺寸（如 52x24），让 CSS 缩放整个 SVG
- **星光虹吸汲取逻辑**：已确认添加 30% 伤害回复逻辑
